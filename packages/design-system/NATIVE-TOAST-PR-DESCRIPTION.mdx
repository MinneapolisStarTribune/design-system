# Technical Description

This PR adds a **native Toast** (`ToastNative`) to the design system so React Native apps can show the same feedback patterns as web: info, success, warning, and error, with optional icon and dismiss control.

Icons match the web Toast (same SVG paths via `react-native-svg`). Types live in shared `Toast.types.ts`. The native package exports `ToastNative`, toast types, and the icon components plus `ToastIconProps` for reuse.

We also export the toast SVG icons from the **native barrel**, refresh the **Getting Started component matrix** so Toast shows as available on native, and include **unit + accessibility tests** for `ToastNative`. Mobile Storybook stories were updated so you can try variants and theme controls on device.

**Note:** Global toast queue / `useToast()` remains **web-only** today. Native apps position and show `ToastNative` themselves (e.g. overlay or inline). **`react-native-svg`** is a normal dependency of this package, so consuming apps do not add it as a separate peer.

## How to test

1. **Automated:** From `packages/design-system`, run `yarn test:native` (and `yarn typecheck` if you want extra confidence).
2. **Mobile Storybook:** Run the mobile Storybook app, open **Feedback & Status → Toast**, and use **Configurable** / **All Variants**; toggle light/dark (or brand) if your preview exposes them.
3. **Quick sanity:** Import `ToastNative` from `@minneapolisstartribune/design-system/native` in an RN app (no extra SVG package step).

## Breaking Changes?

**No** for web consumers.

**Native consumers:** no new peer to declare — **`react-native-svg`** ships as a dependency of the design system. Not an API break.

---

## Risks & Considerations

- **`react-native-svg`** is pinned as a library dependency; duplicate versions in very large monorepos are possible—resolve hoisting if native build issues appear.
- **Snack / global toasts** are not part of this PR — teams that want app-wide stacked toasts on native will layer their own host + state for now.
