# Image

The `Image` component is a reusable wrapper around the native HTML `<img>` element.  
It provides a consistent API for handling images across the application, including support for **Imgix optimization**, **accessibility**, and **testing hooks**.

---

## Canonical usage

```tsx
import { Image } from '@/components/Image';

function Example() {
  return (
    <Image
      src="https://picsum.photos/400/300"
      alt="Sample image"
      width="400"
      height="300"
    />
  );
}
```

---

## With Imgix optimization

```tsx
<Image
  src="https://assets.imgix.net/examples/pione.jpg"
  alt="Optimized image"
  imgixParams="w=400&h=300&fit=crop&auto=format,compress&q=75"
/>
```

---

## Clickable image

```tsx
<Image
  src="https://picsum.photos/400/300"
  alt="Clickable"
  onClick={() => console.log('Clicked')}
/>
```

## Accessibility

- Always provide meaningful <code>alt</code> text.
- The <code>alt</code> attribute is used by screen readers and improves SEO.
- Avoid empty alt text unless the image is purely decorative.

---

## Best practices

- Use **Imgix parameters** to serve optimized images.
- Prefer <code>loading="lazy"</code> for non-critical images.
- Always define width and height to prevent layout shifts.
- Use appropriate formats via Imgix (<code>auto=format</code>).
