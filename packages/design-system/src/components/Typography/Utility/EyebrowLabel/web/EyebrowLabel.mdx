# Eyebrow Label

Eyebrow Labels provide a quick visual signal for story categories, content types, or brand-related context. They appear above headlines on article pages and editorial cards and can be interactive or static depending on their use. In editorial contexts, eyebrows use brand coloring by default to strengthen category recognition, while promotional contexts use neutral eyebrows to keep focus on the promotional content itself. Eyebrows may also indicate special states like live updates or subscriber-only access.

This initial implementation supports:

- three sizes (`small`, `medium`, `large`)
- color variants (**`neutral`**, **`brand`**, **`live`**)
- **`color="live"`** — Applies live attention text color and **prepends a `•` character** before the label as the live indicator (same line as the text; **`aria-hidden`** so screen readers rely on your label wording). No separate prop toggles this dot—the **`live`** color turns it on. The leading brand **`logo`** is hidden whenever **`live`** is used
- optional leading logo: boolean **`logo`** — use **`logo={true}`** (or the JSX shorthand **`logo`**, which means the same thing) to show the brand mark; omit it or set **`logo={false}`** to hide it (default **false**). Hidden when **`color`** is **`live`** or **`isSubscriberOnly`** is true; logo matches text color by default, or override with semantic **`logoColor`** (same tokens as icons)
- optional subscriber-only state via `isSubscriberOnly`: shows a leading key icon and hides the brand logo (same rule as live: no brand mark, only the indicator + text). You can combine `color="live"` with `isSubscriberOnly` for key + live dot + label if needed
- brand-driven logo selection (`startribune` or `varsity`)
- background contrast via `background` (`on-light` or `on-dark`) and legacy `isBackground`

Use this component for short categorical labels (for example section names, promo context, or live states). Keep text concise so the label stays scannable.

Render `EyebrowLabel` inside your app’s **`DesignSystemProvider`** (Storybook does this in the preview) so **`brand`** and theme context resolve when you omit explicit props.

## Usage examples

```tsx
import { EyebrowLabel } from '@minneapolisstartribune/design-system/web';

<EyebrowLabel size="medium" color="neutral" background="on-light">
  Label
</EyebrowLabel>
```

### Subscriber-only key (neutral on light)

Use **`isSubscriberOnly={true}`** to show the **subscriber key** icon before the label. **`logo={true}`** does not show the brand mark in this mode—it follows the same “indicator + label” pattern as **`live`**.

```tsx
import { EyebrowLabel } from '@minneapolisstartribune/design-system/web';

<EyebrowLabel
  size="medium"
  color="neutral"
  background="on-light"
  isSubscriberOnly={true}
  label="Subscriber only"
/>
```
