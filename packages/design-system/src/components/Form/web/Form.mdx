# Form

`<Form>` is an optional layout wrapper that turns our form UI components into a real HTML `<form>`. Use it when you need a semantic form element, native submit behavior, and consistent field and button layout.

## Architecture

The form system is layered — you can use as much or as little as you need:

`FormGroup` and `FormControl` work standalone — drop them in any container and accessibility (`aria-labelledby`, `aria-describedby`) still works correctly. `Form` is what you add when you want a real submittable form.

## Form orientations

### Vertical (default)

Fields stack with 16px gap. The submit button sits at the bottom-right. Use this for most forms: sign-in, sign-up, settings, checkout.

<Canvas of={FormStories.Vertical} sourceState="shown" />

### Horizontal

All inputs and the submit button sit on one row, with labels and descriptions stacked above each input. The bottom edge of every input is aligned with the bottom edge of the button — this holds even when individual groups have descriptions above or error captions below. See the story description below the example for **best for** / **avoid** guidance.

> **How the alignment works:** horizontal layout uses a 3-track CSS subgrid. Each `FormGroup` participates in shared `above` (label + description), `input`, and `caption` row tracks. This means a description in one group or an error caption in another can never push the inputs out of alignment — the input track is shared and fixed.

## Submit with `Form.Button`

Always use `Form.Button` inside `Form` for the submit action. It renders a `type="submit"` button and receives the correct layout treatment for whichever orientation you're using. It accepts the **same props as `Button`** (`color`, `variant`, `size`, and the rest)—use **Button** in Storybook to browse variants and visuals.

Pressing Enter/Return while focused in any input will also submit the form natively, as long as `Form.Button` is present.

## Form logic with `useFormLogic`

`useFormLogic` is an optional hook that gives you controlled field state, validation, and a `handleSubmit` function ready to pass to `<Form>`. Use it when you don't want to wire up form state yourself.

The snippet below is the **body of a component** (not a full file): call the hook, then `return` the `<Form>` and fields.

<Source
  dark
  language="tsx"
  code={`const { values, errors, handleChange, handleBlur, handleSubmit } = useFormLogic({
  initialValues: { email: '' },
  validate: (values) => {
    const errors: Record<string, string> = {};
    if (!values.email) errors.email = 'Email is required';
    return errors;
  },
});

return (
  <Form onSubmit={handleSubmit}>
    <FormGroup>
      <FormGroup.Label>Email</FormGroup.Label>
      <FormControl.TextInput
        name="email"
        value={values.email}
        onChange={(e) => handleChange('email', e.target.value)}
        onBlur={() => handleBlur('email')}
      />
      {errors.email && <FormGroup.Caption variant="error">{errors.email}</FormGroup.Caption>}
    </FormGroup>
    <Form.Button>Submit</Form.Button>
  </Form>
);`}
/>

Skip the hook if you're already using React Hook Form, Formik, or your own `useState` — just pass your own `handleSubmit` to `<Form onSubmit={...}>` directly.

### React Native

`useFormLogic` is platform-agnostic. On React Native there is no `<form>` or DOM change event, so:

- **Field updates:** Use `handleChange(field, value)` instead of `handleChangeEvent`. Wire your text input’s `onChangeText` (or equivalent) to `(value) => handleChange('fieldName', value)`.
- **Submit:** Use `handleSubmit()` with no arguments, or `handleSubmitPress`. Pass the latter to your submit button: `onPress={handleSubmitPress}`. Both run the same validation and `onSubmit` flow.

<Source
  dark
  language="tsx"
  code={`const { values, errors, handleChange, handleBlur, handleSubmitPress } = useFormLogic({
  initialValues: { email: '' },
  validate: (values) => {
    const fieldErrors: Record<string, string> = {};
    if (!values.email) fieldErrors.email = 'Email is required';
    return fieldErrors;
  },
});

return (
  <>
    <TextInput
      value={values.email}
      onChangeText={(value) => handleChange('email', value)}
      onBlur={() => handleBlur('email')}
    />
    <Button onPress={handleSubmitPress} title="Submit" />
  </>
);`}
/>

## Validation

### With validation (simple)

Uses `useFormLogic` for state and validation. Enter an email without the `@` symbol, then blur the field or click Submit to see the error.

**`useFormLogic` options:**

- **initialValues** — Initial field values (keyed by field name).
- **onSubmit** — Called when the form is submitted and validation passes (if `validate` is provided).
- **validate** — Optional. Function that receives current values and returns an object of field names to error messages (omit or undefined = no error).
- **validateOnSubmit** — Run `validate` on form submit. Default: `true`.
- **validateOnBlur** — Run `validate` when a field blurs and set that field’s error. Default: `false`.
