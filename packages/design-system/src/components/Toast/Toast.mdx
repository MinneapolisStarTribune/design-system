# Toast

Toasts surface short, time-sensitive messages without blocking the page. They provide feedback in response to a user action or system event, and are used for confirmations, warnings, and errors that should be easy to scan and dismiss. They appear in a fixed screen location, do not shift layout, and automatically dismiss after 4–6 seconds. Toasts pause on hover or focus and include a manual close button.

[Figma Comps](https://www.figma.com/design/f8DSKwKjG4MsSqmA59v1um/Core-Components)

## Anatomy

1. **Icon (optional)** — Reinforces meaning and aligns with semantic state. Toggle **Show icon** in controls to show or hide it; when visible, the icon matches **Variant** (info, success, warning, error).
2. **Title** — Primary message. Keep it concise, clear, and scannable (**Title** control).
3. **Description (optional)** — Supporting context in one short sentence (**Description** control). Omit when the title alone is enough.
4. **Close button** — Shown by default for manual dismissal (omit with `showCloseButton={false}` only when necessary). The **onClose** handler runs when the user dismisses. With **`ToastRenderer`** + **`useToast()`**, toasts may also auto-dismiss; the inline `Toast` does not implement duration on its own.

## Usage

- Prefer **Variant** for semantic meaning (success after save, error on failure, etc.).
- Keep **Description** to a single sentence when used.
- For global, stacked toasts in a fixed region (e.g. bottom-right), wrap your app with `DesignSystemProvider` and call `useToast()` from `@minneapolisstartribune/design-system/web`.
- In Storybook, provider setup already comes from preview; see **With renderer** for a live hook-driven example.
