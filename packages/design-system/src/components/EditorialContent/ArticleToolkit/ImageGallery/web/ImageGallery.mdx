# Image Gallery

Displays a collection of images with optional captions, credits, caption-driven navigation, and an optional Buy Reprint link in the expanded lightbox view. Supports both standard (inline) and immersive (full-bleed) layouts.

## Usage guidelines

### Variant

- **Standard:** Fills the parent container width — no fixed max-width is applied. Constrain the width at the layout level (e.g. wrap in an article column). Use for inline editorial content.
- **Immersive:** Full-bleed layout with centered active image and adjacent image peek.

### Layout behavior

- Uses CSS aspect ratio to prevent layout shift.
- Images scale responsively with consistent layout.
- Immersive mode centers active slide with adjacent previews.
- Set `aspectRatio` to apply one consistent wrapper ratio across the entire gallery, such as `1 / 1` or `3 / 2`.
- Set `spaceBetween` to override the default slide gap when consumers need custom immersive spacing behavior.

### Image sizing

- Provide `width` and `height` to reserve layout space and prevent CLS.
- Provide `aspectRatio` when the gallery should keep one shared frame regardless of each image's intrinsic dimensions.
- Without `aspectRatio`, the wrapper falls back to the default component styling.

### Imgix support

- Pass `imgixParams` to optimize images (compression, cropping, etc.).
- Example: `w=800&h=500&fit=crop&auto=format,compress`

### Expandable

When `expandable` is enabled, readers can open the **active slide** in a full-screen dialog (same behavior as `InlineImage`). The same shared `Caption` treatment powers inline and lightbox modes.

### Caption-driven gallery controls

- `ImageGallery` delegates inline caption text, credit, counter, and next/previous controls to the shared `Caption` component.
- Use `onIndexChange` to receive the active (1-based) slide index without observing Swiper DOM classes.
- Add `purchaseLink` at the gallery level to share one Buy Reprint CTA across the active image.
- Add `purchaseLink` to each image item (`images[index].purchaseLink`) when a specific image needs its own CTA.
- The Buy Reprint CTA is shown only when the image item's `purchaseLink` includes both `label` and `link`.
- When both are present, the image item purchase link takes precedence.
- In immersive mode, caption navigation stays enabled at the edges and wraps with the looping swiper.

### Custom image component

You can override rendering using `ImageComponent`.

It receives:

- `src`
- `alt`
- `width`
- `height`
- `className`
- All native `<img>` props (via spread)

### Styling hooks

- `className` – root container
- `imageClassName` – image element
- `wrapperClassName` – image wrapper
- `captionClassName` – caption text
- `controlsClassName` – navigation controls

### Accessibility

- Provide meaningful `altText`
- Caption navigation and expand buttons include accessible names
- Expand control uses `aria-haspopup="dialog"`
- Pagination updates are announced through a polite live region
