# Image Gallery

Displays a collection of images with optional captions, credits, Buy Reprint links, and caption-driven navigation. Supports both standard (inline) and immersive (full-bleed) layouts.

## Usage guidelines

### Variant

- **Standard:** Constrained to article width. Use for inline editorial content.
- **Immersive:** Full-bleed layout with centered active image and adjacent image peek.

### Layout behavior

- Uses CSS aspect ratio to prevent layout shift.
- Images scale responsively with consistent layout.
- Immersive mode centers active slide with adjacent previews.

### Image sizing

- Provide `width` and `height` to reserve layout space and prevent CLS.
- Internally uses `aspect-ratio` for better responsive stability.

### Imgix support

- Pass `imgixParams` to optimize images (compression, cropping, etc.).
- Example: `w=800&h=500&fit=crop&auto=format,compress`

### Expandable

When `expandable` is enabled, readers can open the **active slide** in a full-screen dialog (same behavior as `InlineImage`). The same shared `Caption` treatment powers inline and lightbox modes, so the caption, Buy Reprint link, counter, and previous/next controls stay consistent while the dialog is open.

### Caption-driven gallery controls

- `ImageGallery` delegates caption text, credit, Buy Reprint CTA, counter, and next/previous controls to the shared `Caption` component.
- Add `purchaseLink` to an image item to render the optional Buy Reprint CTA inline and in the expanded lightbox view.
- In immersive mode, caption navigation stays enabled at the edges and wraps with the looping swiper.

### Custom image component

You can override rendering using `ImageComponent`.

It receives:

- `src`
- `alt`
- `width`
- `height`
- `className`
- All native `<img>` props (via spread)

### Styling hooks

- `className` – root container
- `imageClassName` – image element
- `wrapperClassName` – image wrapper
- `captionClassName` – caption text
- `controlsClassName` – navigation controls

### Accessibility

- Provide meaningful `altText`
- Caption navigation and expand buttons include accessible names
- Expand control uses `aria-haspopup="dialog"`
- Pagination updates are announced through a polite live region
