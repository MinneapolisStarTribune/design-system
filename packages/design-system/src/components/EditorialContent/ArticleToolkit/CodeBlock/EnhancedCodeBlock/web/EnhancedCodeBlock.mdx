# EnhancedCodeBlock

The **EnhancedCodeBlock** component wraps `DangerousCodeBlock` and applies additional styles for layout variants and sizes.  
It is ideal for article content where styled or interactive code blocks are needed.  
**Security warning:** Only use markup from trusted sources, as this component bypasses React's standard HTML escaping and can execute scripts.

[Figma Comps](https://www.figma.com/file/f8DSKwKjG4MsSqmA59v1um/Core-Components?node-id=1528-6976)

## Usage guidelines

### When to use

- Display code blocks in articles with enhanced layout and styling.
- Show interactive code snippets with scripts or special formatting.
- Choose `standard` or `immersive` variants for article body alignment and emphasis.

### When to avoid

- Avoid if raw HTML is untrusted or user-generated.
- Do not use for simple, plain code snippets where a `pre/code` block suffices.
- Avoid using interactive scripts unless you control the source content.

### Props

- **markup** – the HTML content to render.
- **variant** – controls styling; e.g., `standard` (default) or `immersive`.
- **size** – code block size: e.g., `full`, `large`, `medium`, `inline`.
- **cleanQuotes** – converts smart quotes to straight quotes when `true` (default: `true`).
- **className** – add custom styles.
- **dataTestId** – used for testing; defaults to `enhanced-code-block`.

## React Native usage

`EnhancedCodeBlock` is available from the native entry point and composes `DangerousCodeBlock` with article-layout sizing.

Use it when you need trusted HTML content to respect article-width variants like `full`, `large`, `medium`, or `inline` in mobile layouts.

```tsx
import { EnhancedCodeBlock } from '@minneapolisstartribune/design-system/native';

const markup = `
	<div style="padding: 16px; background: #f4f7fb; border-radius: 8px;">
		<h3 style="margin: 0 0 8px 0;">Enhanced mobile block</h3>
		<p style="margin: 0;">This example applies native size and variant styling.</p>
	</div>
`;

export function NativeEnhancedCodeBlockExample() {
  return <EnhancedCodeBlock markup={markup} variant="standard" size="full" />;
}
```

### Native sizing usage

```tsx
import { EnhancedCodeBlock } from '@minneapolisstartribune/design-system/native';

<EnhancedCodeBlock markup={markup} size="full" />;
<EnhancedCodeBlock markup={markup} size="large" />;
<EnhancedCodeBlock markup={markup} size="medium" variant="immersive" />;
<EnhancedCodeBlock markup={markup} size="inline" />;
```

### Native accessibility and behavior notes

- Native rendering uses the same `WebView`-backed HTML pipeline as `DangerousCodeBlock`.
- `variant` and `size` control the outer React Native layout used around the embedded content.
- `cleanQuotes` is passed through to the underlying dangerous block before rendering.
- The `style` prop can be used to add container spacing or layout overrides.
- Interactive native examples are available in mobile Storybook (`EnhancedCodeBlock.native.stories.tsx`).
