# AuthorBioCard

**AuthorBioCard** displays author information such as name, role, description, avatar, and an optional CTA link.

It is part of the **Editorial / ArticleToolkit** and is used within article pages to present author metadata in a structured and accessible way.

## Usage Guidelines

- Use for **author attribution** in articles
- Supports optional:
  - Avatar image
  - Role/position
  - CTA (profile link / more articles)
- Keeps layout consistent across editorial surfaces


## Content Structure

- **Label** → Section heading (e.g., "Author")
- **Image** → Optional avatar
- **Name** → Author name
- **Position** → Role or designation
- **Description** → Author bio
- **CTA** → Navigation link (profile / articles)


## Variants

### Avatar Shape

- `thumbnailIconRounded = false` → rounded corners (default)
- `thumbnailIconRounded = true` → circular avatar (staff style)


## Borders

- `hasTopBorder` → adds top divider
- `hasBottomBorder` → adds bottom divider

Used for:
- separating multiple authors
- grouping editorial sections


## Accessibility

- Supports:
  - `aria-label`
  - `aria-describedby`
  - `aria-hidden`
- CTA uses semantic **Link** component
- Image requires meaningful `alt` text


## Layout Behavior

- Fixed max-width with responsive adjustments
- Flex layout:
  - Image + content alignment
- Content adapts based on available data

## CTA Behavior

- Uses **Design System Link**
- Supports:
  - `href` → navigation
  - `onClick` → action
- Requires `label` (accessible text)

## How to use
`import { AuthorBioCard , AuthorBioCardProps } from '@minneapolisstartribune/design-system/web'`

## Best Practices

- Always provide **description**
- Provide **alt text** for images
- Avoid empty CTA labels
- Use borders only when needed for separation
