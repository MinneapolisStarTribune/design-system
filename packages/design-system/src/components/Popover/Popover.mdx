# Popover

Popovers provide lightweight, focused surfaces for quick actions or supplemental context. They open on click or focus, can include text, links, and simple actions, and now support richer content through a flexible slot—such as menu items or form controls. Popovers auto-size to their content, reposition as needed, and close when the user clicks outside, presses Escape, or completes an action.

## Anatomy

1. **Container** — The outer surface that holds the popover content and handles positioning and elevation.  
2. **Close button** — Allows users to manually dismiss the popover.  
3. **Title (optional)** — Helps users understand the purpose of the popover.  
4. **Description** — Primary supporting text.  
5. **Header divider (optional)** — Separates header from content when needed.  
6. **Custom content (optional)** — Flexible slot for menus, actions, or forms.  
7. **Pointer** — Directional arrow connecting popover to trigger.

## Usage

- Use for contextual actions or supplementary information
- Keep content concise and scannable
- Use composition (`Popover.Heading`, `Popover.Body`, etc.) for layout control
- Add a divider when separating header and content
- Provide `aria-label` when no heading is present
- Avoid overcrowding content — prefer simple, focused interactions
