# Skeleton

Skeletons are lightweight loading placeholders that hint at the structure of content while data is being fetched. They reduce perceived wait time by showing users where text, images, avatars, or UI elements will appear.

Use skeletons for short, predictable loading moments. Remove them as soon as real content is available — a skeleton that lingers longer than the content it replaces undermines trust.

## Anatomy

A Skeleton renders a single inert block with two shape variants:

1. **Rectangle** (`variant="rectangle"`) — defaults to 200px × 25px. Used for text lines, images, cards, and any block-level content.
2. **Circle** (`variant="circle"`) — defaults to a 64px diameter (`aspect-ratio: 1 / 1`). Used for avatars and icon placeholders.

Both shapes share the same fixed neutral shimmer palette and do not change appearance based on parent background or app theme.

## Usage guidelines

### When to use skeletons

- Content loads in a predictable, short window (under ~3 seconds).
- You know roughly what the layout will look like when content arrives.
- The loading state replaces a specific piece of UI, not the whole page.

### When not to use skeletons

- Loading time is unpredictable or long — prefer a spinner or progress indicator.
- The shape of the loaded content is unknown ahead of time.
- The section is non-critical and can simply render empty.

### Dimensions

Each variant ships with fixed default dimensions. Override with `width` and `height` when the placeholder needs to match specific content:

- **Rectangle** — `200px × 25px` by default. Pass `width` and/or `height` for text lines, images, cards, and other block placeholders.
- **Circle** — `64px × 64px` by default. Pass `width` or `height` to set the diameter; the component keeps a 1:1 aspect ratio.

On web you can also use `className` for layout utilities (e.g. Tailwind `w-32 h-8`) when that fits your app.

There are no preset size tokens (`small` / `medium` / `large`) and no `background` prop — the shimmer uses a **fixed neutral palette** (`#e3e5e8` → `#c8ccd0` on web) that stays the same on light and dark parent surfaces.

### Accessibility

Skeletons are hidden from assistive technology by default (`aria-hidden="true"`). If the loading region needs to be announced, wrap it in a `role="status"` container with an appropriate `aria-label`:

```tsx
<div role="status" aria-label="Loading article">
  <Skeleton variant="rectangle" height={24} />
  <Skeleton variant="rectangle" height={16} width="80%" />
</div>
```

### Animation

The default shimmer animation (`animate={true}`) draws attention to in-progress loading without being distracting. Set `animate={false}` when a parent container already handles animation, or when rendering in a reduced-motion context.

## Composition examples

**Text block placeholder**

```tsx
<div style={{ display: 'flex', flexDirection: 'column', gap: 8 }}>
  <Skeleton variant="rectangle" height={24} />
  <Skeleton variant="rectangle" height={16} />
  <Skeleton variant="rectangle" height={16} width="70%" />
</div>
```

**Author / avatar row**

```tsx
<div style={{ display: 'flex', alignItems: 'center', gap: 12 }}>
  <Skeleton variant="circle" />
  <div style={{ flex: 1, display: 'flex', flexDirection: 'column', gap: 6 }}>
    <Skeleton variant="rectangle" height={12} width="40%" />
    <Skeleton variant="rectangle" height={12} width="25%" />
  </div>
</div>
```

**Article card placeholder**

```tsx
<div style={{ display: 'flex', flexDirection: 'column', gap: 12 }}>
  <Skeleton variant="rectangle" height={180} />
  <Skeleton variant="rectangle" height={24} />
  <Skeleton variant="rectangle" height={16} width="80%" />
</div>
```
