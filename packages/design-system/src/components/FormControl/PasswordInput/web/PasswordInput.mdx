# PasswordInput

`FormControl.PasswordInput` is a masked text field built on top of `TextInput`.

It renders an HTML password input with a keyboard-focusable show/hide toggle, screen reader announcements for visibility changes, and password-manager `autocomplete` hints.

For label, description, and caption composition, use it inside `FormGroup`. This page describes the implemented behavior and public API of `PasswordInput`.

## Canonical usage

```tsx
<FormGroup>
  <FormGroup.Label>Password</FormGroup.Label>
  <FormControl.PasswordInput
    placeholderText="Placeholder text"
    value={password}
    onChange={(e) => setPassword(e.target.value)}
    autoComplete="current-password"
  />
</FormGroup>
```

For sign-up flows, use `autoComplete="new-password"`.

## Visibility toggle

- The input uses `type="password"` when hidden and `type="text"` when visible.
- The toggle is a `<button>` with `aria-label` and `aria-pressed` that update with state.
- `defaultPasswordVisible` sets the initial uncontrolled visibility.
- Use `passwordVisible` and `onPasswordVisibleChange` for controlled visibility.

```tsx
<FormControl.PasswordInput
  value={password}
  onChange={(e) => setPassword(e.target.value)}
  passwordVisible={visible}
  onPasswordVisibleChange={setVisible}
  autoComplete="current-password"
/>
```

When the user toggles visibility, a visually hidden live region announces "Password shown" or "Password hidden" for screen readers.

## Copy and paste behavior

- Paste is always allowed.
- Copy and cut are blocked while the password is hidden.

## Visual options

- `size`: `small` | `medium` | `large`
- `rounded`: rounded corner style
- `isDisabled`: disables input and toggle interaction
- `isError`: applies error styling and `aria-invalid`
- `isSuccess`: applies success styling when not in error

## Accessibility

- Always pass `autoComplete="current-password"` or `autoComplete="new-password"` (WCAG 1.3.5).
- When used inside `FormGroup`, the field automatically inherits:
  - `aria-labelledby` from `FormGroup.Label`
  - `aria-describedby` from `FormGroup.Description` and `FormGroup.Caption`
  - generated field `id` wiring from the form-group context
- When used outside `FormGroup`, provide an accessible name directly, such as `aria-label`.
