# Tooltip

Tooltips provide brief, supplemental information on hover or focus. They’re ideal for clarifying icon-only controls, revealing truncated text, or adding lightweight context that isn’t essential to the primary task. Tooltips should stay short, static, and non-interactive. Use them sparingly, avoid replacing visible labels, and don’t use them for confirmations, warnings, or promotional content.

## Anatomy

1. **Wrapper** — The outer floating container that holds the tooltip content and handles positioning.
2. **Container** — The surface that wraps the content and applies background, border, and styling.
3. **Content** — Flexbox wrapper for label and optional icon.
4. **Label** — Brief supplemental text displayed in the tooltip.
5. **Icon (optional)** — Visual element positioned before or after the label.
6. **Pointer** — Directional arrow connecting tooltip to trigger element.

## Usage

- Use tooltips for clarifying ambiguous UI elements
- Keep tooltip text concise (under 100 characters when possible)
- Use tooltips with icon-only buttons
- Appear on hover and focus, not on click
- Auto-hide when user moves away or presses Escape
- Non-interactive: tooltips never contain buttons, links, or form controls
- Test tooltips on both desktop and mobile

## Example

```tsx
<Tooltip label="Save your changes" pointer="top">
  <Button>Save</Button>
</Tooltip>
```

```tsx
<Tooltip
  label="Required field"
  icon={<WarningIcon />}
  iconPosition="start"
>
  <input type="text" />
</Tooltip>
```

```tsx
<Tooltip label="Settings" pointer="left" showDelay={500}>
  <Button icon={<SettingsIcon />} variant="ghost" />
</Tooltip>
```

## Accessibility

### Keyboard Navigation

- Tooltips appear when the trigger element receives focus
- Users can navigate to tooltip triggers using the keyboard
- Focus is not trapped inside the tooltip
- Escape key dismisses the tooltip

### Screen Readers

- Tooltips use the `tooltip` ARIA role
- Trigger elements have `aria-describedby` pointing to the tooltip
- The tooltip contains an `id` that matches the trigger's `aria-describedby`
- Descriptive label text conveys tooltip purpose to assistive technologies

## Comparison with Popover

| Feature | Tooltip | Popover |
|---------|---------|---------|
| **Trigger** | Hover + Focus | Click |
| **Focus trap** | No | Yes |
| **Interactive content** | No | Yes |
| **Heading/Body** | No | Yes |
| **Close button** | Auto-hide | Yes |
| **Use case** | Quick clarification | Detailed information/actions |
