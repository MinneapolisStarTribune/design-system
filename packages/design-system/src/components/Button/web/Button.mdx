# Button

Buttons trigger user actions and are available in multiple sizes, types, and emphasis levels, with optional icons. Button types define purpose and brand context, while emphasis controls visual weight and presentation.

[Figma Comps](https://www.figma.com/design/f8DSKwKjG4MsSqmA59v1um/Core-Components?node-id=7-3)

## Anatomy

The button is made of a **container**, **label**, and optional **icon**.

1. **Container** — The clickable or tappable frame that defines the button’s shape, size, and hit area. It uses fixed corner radius and size tokens, with padding scaled to each size.
2. **Label** — A short, single-line text description. The text should not wrap or truncate.
3. **Icon (optional)** — Can sit on the left or right of the label. It uses fixed icon sizes and spacing and inherits the label color.

Use **Configurable** in the stories below to explore props (including **`surface`**) — open the **Controls** tab in the Storybook addons panel. **On dark background** demonstrates **`surface="dark"`** on a dark module. **All variants** shows the full matrix.

## Usage guidelines

### Color

- **Neutral:** The standard button style used for most actions across the product.
- **Brand:** Brand buttons are used for high-priority calls to action, such as subscription prompts. They help reinforce brand identity and should be used sparingly to preserve visual hierarchy and emphasis. Use brand buttons when an action is critical to business goals and needs to stand out from neutral UI.
- **Brand-Accent:** Brand accent buttons are a specialized variant of brand buttons and are intended only for use on dark brand backgrounds to maintain proper contrast and brand impact. Because of their strong visual weight, brand accent buttons should be used very selectively.

### Variant

- **Filled:** Filled buttons use a solid background to provide strong visual presence. They work well when an action needs to be clearly distinguishable from surrounding content.
- **Outlined:** Outline buttons use a stroked container with minimal fill. They are useful in layouts where a lighter visual treatment is preferred or when pairing multiple actions.
- **Ghost:** Ghost buttons use minimal styling and rely primarily on text and subtle interaction states. They are well suited for situations where visual weight should be kept low or when pairing multiple actions.

### Size

- **Small:** Used in dense layouts, mobile UI, or anywhere vertical space is limited.
- **Medium:** Standard size for most UI actions across both brands.
- **Large:** High-impact option used primarily in marketing or promotional contexts.

### Icons

Icon Buttons offer a compact way to surface common actions. Use them for recognizable, quick actions (like close, add or share) or anywhere a lighter, space-efficient control is needed.

### Surface

Use **`surface`** when the button sits on a **dark region** (for example a candy bar, dark card, or black bar) while the page still loads **light** theme CSS on `:root`. In that case, default button tokens can lack contrast; **`surface="dark"`** applies the same on-dark button palette you would get in global dark mode, scoped to this control only.

- **`light`** (default) — Standard tokens from the active light theme on `:root`.
- **`dark`** — Dark-surface button colors. **Neutral** tokens are shared. **Brand** and **brand-accent** follow the product brand set on **`DesignSystemProvider`**, reflected on `<html>` as **`data-ds-brand`** (for example `varsity` vs Star Tribune). No extra prop is required beyond `surface`.

## Next.js App Router

Use the default **`<button>`** when the control **performs an action** (submit, open, toggle) in place. Use **`as="a"`** with **`href`** for a **plain anchor** (external URL or when you do not need client-side routing). Use **`next/link`** as **`as`** when you want **button styling** with **App Router** navigation (client transitions, **`prefetch`**, **`replace`**, **`scroll`**, etc.).

```tsx
import NextLink from 'next/link';
import { Button } from '@minneapolisstartribune/design-system/web';

<Button as={NextLink} href="/subscribe" prefetch={false} variant="filled" color="brand" size="medium">
  Subscribe
</Button>
```

**Appearance:** Same as any other **`Button`** (variants, sizes, radius). **`NextLink`** only swaps the root to an anchor for routing — not the typography **`Link`** component (underline / inline styles).

**How it works:** The design system does **not** import `next` — you pass whatever component should render at the root (here **`NextLink`**). Standard **`Button`** props and **`next/link`** props you pass are **forwarded** to that root. When the root is **not** a native **`<button>`**, **`disabled`** is expressed with **`aria-disabled`** and pointer/keyboard interaction is blocked (anchors do not use the invalid **`disabled`** attribute).
