## Using breakpoints in media queries

> **Heads up:** CSS custom properties (`var(--breakpoint-*)`) **cannot** be used inside `@media` query conditions. Media features are evaluated before the cascade runs, so the browser has no element context to resolve a custom property against and silently treats the rule as invalid. This is a CSS spec limitation, not a browser bug.
>
> The design system still exposes `--breakpoint-large-min`, `--breakpoint-medium-min`, etc. as CSS variables — they're useful in regular property values (e.g. `max-width: var(--breakpoint-large-container-max)`) but **not** inside `@media (...)`.

In SCSS, declare the breakpoint values as Sass variables (resolved at compile time) and use those in your media queries. Keep them in sync with the values in the table above.

```scss
$breakpoint-small-max: 767px;
$breakpoint-medium-min: 768px;
$breakpoint-medium-max: 1159px;
$breakpoint-large-min: 1160px;

@media (min-width: $breakpoint-large-min) {
  /* Large breakpoint styles */
}

@media (min-width: $breakpoint-medium-min) and (max-width: $breakpoint-medium-max) {
  /* Medium breakpoint styles */
}

@media (max-width: $breakpoint-small-max) {
  /* Small breakpoint styles */
}
```

In plain CSS, use the literal pixel values directly:

```css
@media (min-width: 1160px) {
  /* Large breakpoint styles */
}

@media (min-width: 768px) and (max-width: 1159px) {
  /* Medium breakpoint styles */
}

@media (max-width: 767px) {
  /* Small breakpoint styles */
}
```
