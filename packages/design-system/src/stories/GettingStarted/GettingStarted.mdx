# Getting Started

Welcome to the Minneapolis Star Tribune Design System Storybook. This is the
canonical reference for our shared UI components, tokens, and patterns across
web and native experiences.

## How to use this Storybook

- **Foundations**: Explore colors, typography, spacing, and other tokens under
  the `Foundations` section. These pages describe the building blocks behind
  every component.
- **Components**: Browse UI components grouped by category (e.g., Actions,
  Inputs, Typography, Feedback). Each component has a **Docs** tab with
  guidance and live examples.
- **Utilities & Stories**: Some top-level stories showcase utilities,
  icon galleries, or testing views used for Chromatic and a11y.

## Reading a component page

Each component Docs page typically includes:

- **Definition & usage guidelines** written by UX/design.
- **Configurable example** allowing a full playground of what is possible for this component.
- **Controls / Props table** where you can tweak args and see how the component
  responds.
- **All Variants Story** to see all variantions and combinations of this component.

## Using components in code

- **Web**: Import from `@minneapolisstartribune/design-system/web` and use the
  provided React components. Most stories map directly to props you can use in
  product code.
- **Native**: Import from `@minneapolisstartribune/design-system/native` for
  React Native implementations where available.

Refer to your existing integration guides (analytics, theming, etc.) for
environment-specific setup and wiring.

## Who this is for

- **Designers**: To define and evolve component guidelines, usage patterns, and
  examples via MDX.
- **Engineers**: To consume components consistently, discover APIs, and verify
  behavior.
- **Product & QA**: To see what’s available out of the box and how it should
  look and behave across brands and themes.

For a platform-by-platform overview of what’s available, see the
**Getting Started / Component Matrix** page.
